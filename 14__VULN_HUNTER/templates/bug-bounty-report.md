## Description

Hardcoded AWS Access Key and Secret Key were discovered in the youtube-dl source code repository. These credentials are embedded directly in the `shahid.py` extractor module and are used for AWS API operations within the `_aws_execute_api` function.

The credentials allow authenticated access to AWS services and could be exploited by attackers to:
- Access AWS infrastructure under the compromised account
- Spin up expensive resources (cryptocurrency mining, botnets)
- Access sensitive data stored in AWS
- Pivot to other internal systems

## Remediation

1. **Immediately rotate** the exposed AWS Access Key (AKIAI6X4TYCIXM2B7MUQ)
2. **Remove** hardcoded credentials from source code
3. **Use environment variables** or a secrets manager (AWS Secrets Manager, HashiCorp Vault)
4. **Implement least-privilege IAM policies** for AWS operations
5. **Enable AWS CloudTrail** logging to monitor for unauthorized use
